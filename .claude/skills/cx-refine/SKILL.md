# Skill: cx-refine

## Description
Iteratively refine documents, specs, and code through structured review cycles. Applies quality criteria and suggests improvements.

## Triggers
- Developer asks to improve or polish a document
- Spec review is requested
- Code quality review is needed

## Steps
1. Read the target document or code
2. Apply quality criteria relevant to the content type
3. Suggest specific, actionable improvements
4. Iterate with the developer until satisfied

## Rules
- Always explain why a change is suggested
- Respect the developer's voice and style
- Focus on substance over formatting
- One round of refinement at a time
